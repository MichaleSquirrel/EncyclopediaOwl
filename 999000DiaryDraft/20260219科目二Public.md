3、（多选题）在将集合List<string> names (集合不为空)转换为数组时，在节省空间和效率方面，Java语言编程规范推荐的方式有（AD ）

A、

String[] nameArray = names.toArray(String[]::new);
B、

String[] nameArray = names.toArray(new String[names.size() - 1]);
C、

String[] nameArray = names.toArray(new String[names.size()]);
D、

String[] nameArray = names.toArray(new String[0]);
解析：
Java 11引入了Collection<T>.toArray(IntFunction<T[]> generator)，它更好的原因是不需要创建临时数组，一方面节省空间，另一方面这样就不用去考虑toArray(T[])里的参数长度对方法行为以及结果的影响。
【正例】（Java 11+）
List<String> xs = ...;
String[] sa = xs.toArray(String[]::new);
另外，java.util.stream中各Stream的toArray()、toArray(IntFunction<A[]>)也是常用的。
Java 11前toArray(T[] a)的参数应采用零长度的数组，这样可保证有更好的性能。数组容量大小产生的影响如下：

等于0，动态创建与size相同的数组，性能最好；
大于0但小于size，重新创建大小等于size的数组，增加GC负担；
等于size，在高并发情况下，数组创建完成之后，size正在变大的情况下，负面影响与上相同；
大于size，空间浪费，且在size处插入null值，存在NullPointerException隐患。
【正例】（Java 11-）

List<String> list = new ArrayList<>(DEFAULT_CAPACITY); list.add(getElm()); String[] array = list.toArray(new String[0]);

问题分析
```text
1. `Base base = new Child();`：创建了一个 `Child` 类的对象，并将其引用赋值给 `Base` 类型的变量 `base`。这称为向上转型。

2. `System.out.println(base.id);`：访问 `base` 对象的 `id` 属性。由于 `id` 是实例变量，而不是方法，因此不会发生多态。在向上转型的情况下，访问的是 `Base` 类的 `id` 属性，而不是 `Child` 类的 `id` 属性。因此，输出 `100`。

3. `base.doSomething();`：调用 `base` 对象的 `doSomething` 方法。由于 `doSomething` 是一个方法，会发生多态。虽然 `base` 是 `Base` 类型的引用，但实际对象是 `Child` 类的对象，因此调用的是 `Child` 类重写后的 `doSomething` 方法，输出 `Child`。
```

问题分析
```text
switch case 语句有如下规则：
switch 语句中的变量类型可以是： byte、short、int 或者 char及其包装类和Enum。从 Java SE 7 开始，switch 支持字符串 String ，同时 case 标签必须为字符串常量或字面量。
switch 语句可以拥有多个 case 语句。每个 case 后面跟一个要比较的值和冒号。
case 语句中的值的数据类型必须与变量的数据类型相同，而且只能是常量或者字面常量。
当变量的值与 case 语句的值相等时，那么 case 语句之后的语句开始执行，直到 break 语句出现才会跳出 switch 语句。
当遇到 break 语句时，switch 语句终止。程序跳转到 switch 语句后面的语句执行。case 语句不必须要包含 break 语句。如果没有 break 语句出现，程序会继续执行下一条 case 语句，直到出现 break 语句。
switch 语句可以包含一个 default 分支，该分支一般是 switch 语句的最后一个分支（可以在任何位置，但建议在最后一个）。default 在没有 case 语句的值和变量值相等的时候执行。default 分支不需要 break 语句。
```

> 字符流 和 字节流 区别


字节流和字符流是Java I/O系统中两种核心的数据处理方式，它们在数据单位、编码处理、适用场景等方面存在显著差异。

核心差异对比

| 特性 | 字节流 | 字符流 |
| :--- | :--- | :--- |
| 数据处理单位 | 以字节（8位二进制）为单位处理数据。 | 以字符（Java中为16位Unicode字符）为单位处理数据。字符的实际字节数取决于编码（如UTF-8中一个字符占1-4字节）。 |
| 编码处理 | 不涉及字符编码，直接读写原始字节数据。处理文本时，若编码不匹配，极易出现乱码。 | 自动处理字符编码。在读写时，会根据指定的编码（如UTF-8、GBK）进行字节与字符之间的自动转换，能有效避免文本乱码问题。 |
| 基类与常用实现 | 抽象基类为 `InputStream` 和 `OutputStream`。常用实现类：`FileInputStream`、`FileOutputStream`、`BufferedInputStream`、`BufferedOutputStream`。 | 抽象基类为 `Reader` 和 `Writer`。常用实现类：`FileReader`、`FileWriter`、`BufferedReader`、`BufferedWriter`。 |
| 缓冲机制 | 默认不使用缓冲区。若需提高性能，需手动包装 `BufferedInputStream` 或 `BufferedOutputStream`。 | 默认使用缓冲区。例如 `BufferedReader` 和 `BufferedWriter` 在内部自动管理缓冲，能显著提升文本读写效率。 |
| 主要适用场景 | 适用于处理所有类型的二进制文件，如图片（JPG、PNG）、音频（MP3）、视频（MP4）、压缩包（ZIP）等。也适用于需要精确控制数据的场景。 | 专为处理文本数据设计，如 `.txt`、`.java`、`.xml`、`.json` 等文件。在处理包含中文、日文等非ASCII字符的文本时，是首选。 |
| 底层关系 | 是基础的数据流类型。 | 底层依赖于字节流。字符流（如 `InputStreamReader`、`OutputStreamWriter`）本质上是字节流的包装器，通过添加编码转换层来实现字符处理。 |

总结
选择使用字节流还是字符流，关键在于处理的数据类型：
*   处理二进制文件或需要精确控制字节 → 选择字节流。
*   处理文本文件，追求便捷与避免乱码 → 选择字符流。

在实际开发中，字符流因其对编码的自动处理和内置缓冲，通常更方便、更安全地处理文本数据。
